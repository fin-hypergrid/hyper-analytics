Class: DataSourceTreeview

DataSourceTreeview

For proper sorting, include DataSourceTreeviewSorter in your data source pipeline, ahead of (closer to the data than) this data source.

For proper filtering, include DataSourceTreeviewFilter in your data source pipeline, ahead of DataSourceTreeviewSorter, if included; or at any rate ahead of this data source.

click(y, expandopt, depthopt) → {undefined|boolean}

Handle a click event in the drill-down column.

Operates only on the following rows:

  • Expandable rows - Rows with a drill-down control.
  • Revealed rows - Rows not hidden inside of collapsed drill-downs.
Parameters:
Name Type Attributes Default Description
y

Revealed row number. (This is not the row ID.)
expand boolean <optional>

One of:

  • true - Expand all rows that are currently collapsed.
  • false - Collapse all rows that are currently expanded.
  • undefined (or omitted) - Expand all currently collapsed rows; collapse all currently expanded rows.
depth number <optional>
 Infinity

One of:

  • number > 0 - Apply only if row depth is above the given depth.
  • number <= 0 - Apply only if row depth is below the given depth.
Returns:

  • undefined - Row was not expandable.
  • true - Row had drill-down and state changed.

setRelation(optionsopt) → {boolean}

Toggle the tree-view.

Calculates or recalculates nesting depth of each row and marks it as "expandable" iff it has children.

If resetting previously set data, the state of expansion of all rows that still have children is retained. (All expanded rows will still be expanded when tree-view is turned back ON.)

All of the columns referenced by the options properties idColumn, parentIdColumn, treeColumn, and groupColumn must exist. These four columns have default references (names) as listed below. The references may be overridden in options by supplying alternate column names or indexes.

Parameters:
Name Type Attributes Description
options boolean | object <optional>

Falsy value (or omitted) turns tree-view OFF. Truthy value turns tree-view ON using following options:

See http://openfin.github.io/fin-hypergrid/tree-view-separate-drill-down.html for an example.
